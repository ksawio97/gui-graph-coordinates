## Wymagania

### Graficzny interfejs użytkownika

Projekt w Javie posiada graficzny interfejs użytkownika wykonany w technologii SWING. Interfejs graficzny powinien składać się z:

- paska menu z (co najmniej) opcją do: wczytania grafu z pliku tekstowego i binarnego oraz zapisu wyników oraz, ewentualnie, także grafu,
- komponentu prezentującego wczytany graf (należy zaimplementować samodzielnie metodę do rysowania tego komponentu SWING),
- dodatkowego panelu narzędziowego pozwalającego na: wybór algorytmu, modyfikację współrzędnych, zmianę pola widzenia, zmianę sposobu wyświetlania węzłów/krawędzi (np. pokazywanie/ukrywanie etykiet/wag).

Proszę zwrócić uwagę na informowanie użytkownika o błędach oraz o zachowanie spójności między stanami komponentów.

W trakcie semestru może nastąpić niewielkie zmiany/doprecyzowanie niektórych wymagań, co ma pokazać, że tworzony program jest elastyczny i pozwala na wprowadzenie zmian
### Dokumentacja projektu Java

- Projekt interfejsu użytkownika w Java z wykorzystaniem Java Swing - instrukcja użytkownika.
- Podział systemu na moduły, główne wzorce projektowe.
- Dokumentacja końcowa projektu (z diagramem klas).

W ramach dokumentacji końcowej należy zawrzeć pełen opis projektu﻿﻿ (analogicznie jak w części z języka C) rozszerzony o pełen diagram wykorzystanych klas (dopuszczalne wykorzystanie narzędzi pomocniczych do generowania diagramu) oraz informacji koniecznych do przekazania zespołowi programistycznemu. W diagramie klas nie należy uwzględniać klas systemowych chyba, że wymaga tego opis wykorzystywanej funkcjonalności.
